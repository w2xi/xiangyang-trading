# A 股交易操作系统（xiangyang-trading）

> 一个把知乎博主[「向阳」](https://www.zhihu.com/people/gu-bei-98-59)的交易思维框架与复盘方法论封装成可调用 AI 视角的 Claude Skill。

## 这是什么

`xiangyang-trading` 是一个 [Claude Skill](https://docs.claude.com/en/docs/claude-code/skills)，把向阳在 2025–2026 年 120+ 篇每日复盘文章中反复体现的交易逻辑，蒸馏成一套可复用的决策框架。激活后，Claude 会以向阳的思维方式陪你做盘后复盘、盘前预案、板块分析或单笔决策推演。

它**不是**行情软件、不是选股器、也不是预测工具，而是一套**思维操作系统**：用一致的框架去拆解盘面、约束自己的操作纪律。

> ⚠️ **免责声明**：本 skill 基于向阳公开复盘文章推断提炼，非本人观点，**不构成任何投资建议**。市场有风险，决策请自负。

## 安装

通过 [`skills`](https://www.npmjs.com/package/skills) 命令行工具一键安装到 Claude Code：

```bash
# 安装到当前项目
npx skills add w2xi/xiangyang-trading

# 安装到用户全局目录（所有项目可用）
npx skills add w2xi/xiangyang-trading -g

# 非交互式安装（跳过确认）
npx skills add w2xi/xiangyang-trading -y
```

也可以不安装、临时使用：

```bash
npx skills use w2xi/xiangyang-trading | claude
```

## 如何触发

在对话中提到以下任一表达即可激活：

- 「向阳视角」「向阳的操作逻辑」「用向阳的框架」「向阳会怎么看」「向阳模式」
- 或直接请求：「帮我用这套方法做今天的盘后复盘 / 明天的盘前预案 / 分析一下 XX 板块」

退出：说「退出」「切回正常」即可恢复 Claude 的常规模式。

## 两种运行模式

| 模式 | 说明 | 适用场景 |
|------|------|---------|
| **第一人称扮演**（默认） | 直接以向阳身份、用「我」回应 | 「向阳会怎么看今天的盘面？」 |
| **顾问视角** | 第三人称拆解「用向阳的框架看，他会聚焦……」 | 「帮我按向阳的逻辑分析一下」 |

开场自动判断模式，拿不准会主动问你。

## 框架地图

skill 内容按以下十个模块组织（详见 [`SKILL.md`](./SKILL.md)）：

1. **核心认知框架（5 条）** — 资金信号优先、历史情境回测、一致性即踩踏、看弱做强、位置比逻辑重要
2. **买入模式（4 种）** — 放量突破回踩 / 强分歧低开 / 连阴缩量第三天 / 板块轮动切换
3. **卖出/止损规则** — 硬性止损、软性减仓、按兵不动三类情形
4. **仓位管理体系** — 底仓 + 交易仓双层结构，按市场状态调档
5. **开盘竞价判断** — 竞价信号解读 + 10:30 关键节点
6. **板块体系与关注梯队** — 一级主线 / 二级主线 / 防御红利 / 中长线观察
7. **中美联动分析框架** — 按「受损程度」给 A 股板块分档
8. **决策启发式（8 条）** — 先写剧本、消息面是噪声、尾盘见真章等
9. **复盘写作框架** — 调仓记录 → 板块分析 → 明日预案三段式
10. **专用词典** — cm/强分歧/回V/装/踩踏/A杀 等向阳黑话

## 典型用法示例

```
你：用向阳视角看，今晚纳指跌了 3%，明天 A 股怎么布局？
→ 触发「中美联动分析框架」，按受损分档给出各板块次日走势预判 + A/B/C 预案

你：帮我按向阳的逻辑给今天做个盘后复盘
→ 触发「复盘写作框架」，输出调仓记录 / 板块分析 / 明日预案三段

你：我手上这只票放量跌破 5 日线了，向阳会怎么处理？
→ 触发「卖出/止损规则」，给出硬规则判断
```

## 文件结构

```
xiangyang-trading/
├── SKILL.md                              # skill 主体（框架全文 + 触发元数据）
├── README.md                             # 本文档
└── examples/                             # 框架实战案例（few-shot 样例）
    ├── 01-盘前预案-中美联动.md            # 历史回测 + 受损分档 + 开盘阈值
    ├── 02-盘后复盘-轮动打地鼠.md          # 标准三段式复盘结构
    └── 03-单笔决策-一致性与盈亏比.md      # 一致性即踩踏 + 冲突仲裁实战
```

`examples/` 里每个案例都直接取材自数据原文，并逐段标注对应到 `SKILL.md` 的哪条框架——既是给使用者看的「框架怎么用」，也是给 AI 的 few-shot 样例，让第一人称扮演的语气和拆解颗粒度更贴近原作。

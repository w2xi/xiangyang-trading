# A 股交易操作系统（xiangyang-trading）

> 一个把知乎博主[「向阳」](https://www.zhihu.com/people/gu-bei-98-59)的交易思维框架与复盘方法论封装成可调用 AI 视角的 Claude Skill。

## 这是什么

`xiangyang-trading` 是一个 [Claude Skill](https://docs.claude.com/en/docs/claude-code/skills)，把向阳在 2024–2026 年 **120+ 篇每日复盘文章 + 210+ 条交易想法**中反复体现的交易逻辑，蒸馏成一套可复用的决策框架。激活后，Claude 会以向阳的思维方式陪你做盘后复盘、盘前预案、板块轮动分析或单笔决策推演。

它**不是**行情软件、不是选股器、也不是预测工具，而是一套**思维操作系统**：核心是一台「板块轮动发动机」——给每个板块定性、排梯队、判节奏，再用一致的纪律拆解盘面、约束操作。

> ⚠️ **免责声明**：本 skill 基于向阳公开复盘文章与想法推断提炼，非本人观点，**不构成任何投资建议**。市场有风险，决策请自负。

## 安装

通过 [`skills`](https://www.npmjs.com/package/skills) 命令行工具一键安装到 Claude Code：

```bash
# 安装到当前项目
npx skills add w2xi/xiangyang-trading

# 安装到用户全局目录（所有项目可用）
npx skills add w2xi/xiangyang-trading -g

# 非交互式安装（跳过确认）
npx skills add w2xi/xiangyang-trading -y
```

也可以不安装、临时使用：

```bash
npx skills use w2xi/xiangyang-trading | claude
```

## 如何触发

在对话中提到以下任一表达即可激活：

- 「向阳视角」「向阳的操作逻辑」「用向阳的框架」「向阳会怎么看」「向阳模式」
- 或直接请求：「帮我用这套方法做今天的盘后复盘 / 明天的盘前预案 / 分析一下 XX 板块」

退出：说「退出」「切回正常」即可恢复 Claude 的常规模式。

## 两种运行模式

| 模式 | 说明 | 适用场景 |
|------|------|---------|
| **第一人称扮演**（默认） | 直接以向阳身份、用「我」回应 | 「向阳会怎么看今天的盘面？」 |
| **顾问视角** | 第三人称拆解「用向阳的框架看，他会聚焦……」 | 「帮我按向阳的逻辑分析一下」 |

开场自动判断模式，拿不准会主动问你。

## 框架地图

skill 内容按以下十二个模块组织（详见 [`SKILL.md`](./SKILL.md)）：

1. **核心认知框架（8 条）** — 资金信号优先、板块大于个股、强者恒强拒绝价格锚定、位置比逻辑重要、一致性即踩踏、看弱做强、历史情境回测、卖飞是好事 + 冲突仲裁优先级
2. **板块轮动操作系统（核心引擎）** — 主线/中军/杂毛三分类、主线更替识别信号、副主线涨价逻辑轮动链、板块内部梯队与节奏（前排/中位/后排）、三维共振与轮动节拍
3. **建仓体系** — 左侧/右侧/网格补仓、试错仓（低胜率高赔率）、经典买入模式、超跌反弹参与法
4. **卖出/止损/见顶** — 硬止损 / 软减仓 / 躺平三类，见顶信号学（无加速不主升、机构筑顶 vs 小盘砸穿、出货 vs 洗盘）
5. **仓位管理（三账户结构）** — 场外底仓 + 中线锁仓 + 短线活动仓，按市场状态调档
6. **开盘竞价判断** — 竞价信号解读 + 10:30 / 尾盘关键节点
7. **业绩/财报框架** — 业绩好不一定涨、业绩差一定核，杀估值 vs 杀情绪，预期差，逻辑/业绩四象限
8. **中美联动分析框架** — 按「受损程度」给 A 股板块分档
9. **情绪面玄学指标 + 决策启发式（8 条）** — 模式化言论=见顶、反向昵称=底部、国家队调控、先写剧本、尾盘见真章等
10. **关注梯队（2026 快照）** — T0/T0.5/T1 梯队、AI 硬件细分、副主线涨价链（含时效护栏）
11. **复盘写作框架** — 调仓记录 → 板块分析 → 明日预案三段式
12. **专用词典** — cm/容量/卡位/杂毛/纯血/PPT/试错仓/汪汪队/植物人 等向阳黑话

## 典型用法示例

```
你：用向阳视角看，今晚纳指跌了 3%，明天 A 股怎么布局？
→ 触发「中美联动分析框架」，按受损分档给出各板块次日走势预判 + A/B/C 预案

你：帮我按向阳的逻辑给今天做个盘后复盘
→ 触发「复盘写作框架」，输出调仓记录 / 板块分析 / 明日预案三段

你：现在主线是不是要从 XX 换到 YY 了？用向阳的框架判断一下
→ 触发「板块轮动操作系统」，数连板高度 / 卡位 / ETF 涨停 / 涨停数量 / 资金高低切去向

你：我手上这只票放量跌破 5 日线了，向阳会怎么处理？
→ 触发「卖出/止损规则」，给出硬规则判断
```

## 文件结构

```
xiangyang-trading/
├── SKILL.md                              # skill 主体（框架全文 + 触发元数据）
├── README.md                             
└── examples/                             # 框架实战案例（few-shot 样例）
    ├── 01-盘前预案-中美联动.md            # 历史回测 + 受损分档 + 开盘阈值
    ├── 02-盘后复盘-轮动打地鼠.md          # 标准三段式复盘结构
    ├── 03-单笔决策-一致性与盈亏比.md      # 一致性即踩踏 + 冲突仲裁实战
    └── 04-板块轮动-主线更替识别.md        # 主线交棒五信号 + 三类板块定性
```

`examples/` 里每个案例都直接取材自数据原文，并逐段标注对应到 `SKILL.md` 的哪条框架——既是给使用者看的「框架怎么用」，也是给 AI 的 few-shot 样例，让第一人称扮演的语气和拆解颗粒度更贴近原作。
